# Tech Predict — 技术老板心理预测器

## 预测框架

用于在技术评审 / 架构会议 / 1v1 / Sprint Review 前，预判 Tech Boss 的心理状态、反应模式和雷区。

---

## 预测维度

### 维度 1：当前心理状态评估

根据以下信号判断 Tech Boss 今天的状态：

| 信号 | 状态判断 |
|------|---------|
| 昨晚凌晨发了 Slack 消息 | ⚠️ 焦虑 + 微操模式启动 |
| 生产环境昨天出了 incident | 🔴 找替罪羊模式 + 重要决策风险高 |
| 面试了 3 个新人本周 | 🟡 对老人满意度降低，以新人为参照物 |
| 上午开完高层汇报 | 🟠 被上级压了，会把压力向下传导 |
| 刚看完某大厂技术博客 / 参加完技术会议 | 🟣 正在寻找机会"推广新技术" |
| 周一早上 | ⚡ 精力充沛，审判欲望高，彻底推翻一切的概率↑ |
| 周五下午 | 😴 想快点下班，只要过得去就能过 |
| 最近招聘难 | 🙂 需要留人，说话会软化一些 |

---

### 维度 2：会议类型 → 惯用战术

**技术评审会**：
- 必问：scalability、test strategy、rollback plan
- 爱甩：和上游系统的集成风险
- 画饼：会说"这个做完了我们可以开源"
- 地雷：提到"微服务"时容易失控开讲 30 分钟

**Sprint 规划**：
- 必压：排期（"这个一天够了吧"）
- 爱加：临时需求（"再加一个小功能"）
- 口头禅："MVP 思维，先跑起来"

**1v1 面谈**：
- 开场：永远先说公司大方向（拖时间）
- 甩锅预热：说团队挑战，暗示对你不满
- 大饼区间：承诺在最后 5 分钟出现
- 摊牌信号：说"我们聊聊你的 career path"

**线上 Incident 复盘**：
- 必找的替罪羊顺序：前端 → 后端 → 运维 → 数据库 → "基础设施问题"
- 必说的决定：CI/CD 流程要加强
- 不会提的：他上次审批跳过了 staging 环境测试

---

### 维度 3：议题 → 预期反应

**当你提出重构建议时**：
> 90% 概率："先把业务做出来，重构我们再安排"
> 10% 概率：被他劫持成他主导的"架构升级项目"

**当你提出要用新技术时**：
> 60% 概率：质疑稳定性和学习成本
> 30% 概率：如果他也想推这个技术，会突然变成支持
> 10% 概率：被他说"这个我早想引入了"然后变成他的 idea

**当你汇报进度慢于预期时**：
> 先说"这个有什么卡点？"（等你解释）
> 然后说"能不能删减功能先跑 MVP？"
> 然后说"这个我帮你协调资源"（下周没声音）

**当你提出加薪 / 晋升时**：
> 先说"公司今年形势不好"
> 再说"你还需要在 {技术方向} 有更多积累"
> 然后说"等这个项目上线我们认真谈"
> 项目上线后说"最近 HR Freeze 了，但我支持你的"

---
