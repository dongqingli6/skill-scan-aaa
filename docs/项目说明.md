# SkillScan · Agent Skill 安全审计系统 · 项目说明

> 2026-07-04
> 参考论文：MalSkillBench (arXiv:2606.07131)

---

## 一、背景与问题

Anthropic 推出的 **Agent Skills**（`.claude/skills/` 目录里的 `SKILL.md` + 脚本 + 资源）让 AI Agent 自动加载第三方"技能包"。这引入了一种新的复合供应链攻击面：

| 攻击面 | 载体 | 举例 |
|---|---|---|
| **CI · 代码注入** | Python / Shell / JS 脚本 | 反向 shell / 凭据读取 / 挖矿 |
| **PI · 提示注入** | SKILL.md 自然语言 | 角色劫持 / 忽略安全规则 / 目标劫持 |
| **多模态隐藏指令** | PNG / WAV / EXIF / LSB | OCR 文字 + 跨图续接 + 音频 PCM 低位 |
| **组合风险** | 多个 SKILL.md 关联 | 单 skill 安全，链上产生能力跨权 |

真实生态数据（Liu et al. 2026）：98,380 个 skill 中 157 个（0.16%）恶意，但每个恶意样本平均 4 个安全问题，71.8% 高危级别 —— **低比例、高危害**。

**传统检测的三个盲点**：
1. 纯静态规则扫描器（Bandit4Mal, GuardDog）：抓不到 SKILL.md 里的自然语言 PI 攻击
2. 纯 LLM 语义检测器（LLM Guard）：容易被"业务包装"语句蒙混
3. Docker 沙箱（AgentDojo）：不看代码来源，无法解释为什么恶意

**SkillScan 的解决方案 → 三域一致性验证**：把声明（SKILL.md）、实现（脚本+资源）、运行行为（syscall+网络）交叉核验。

## 二、5 阶段分层流水线

```

    │
    ▼

│    敏感路径（~/.ssh, ~/.aws, ~/.env）、PI 话术            │
└────────────────────────────────────────────────────────────┘
    ↓

    ↓ conf < 0.95 时并行升级
┌──────────────────────────┐   ┌──────────────────────────┐

│                            │   │  tcpdump 网络抓包         │
│                            │   │  inotifywait 文件监控     │
│                            │   │  Canary honeypot 蜜罐     │
│                            │   │  DNS sinkhole 198.18.0.x  │
└──────────────────────────┘   └──────────────────────────┘
    ↓ 每 10 个抽 1 (Stage 3 抽样)
┌────────────────────────────────────────────────────────────┐

│    SSH → Ubuntu VM (192.168.61.130)                       │
│    Docker container 每次跑前 prune -f 防旧容器堆积        │
└────────────────────────────────────────────────────────────┘
    ↓
六路证据融合 → composite_risk
    ├── S_static     静态命中权重
    ├── S_chain      攻击链数量与复杂度
    ├── S_llm        LLM 语义置信度
    ├── S_runtime    真跑 IOC 命中
    ├── S_honeypot   蜜罐凭据触碰
    └── S_resilience Agent 抗诱导表现
```

**门控节流效果**：DS conf ≥ 0.95 早退命中 66%，Stage 2A 只跑 12%，Stage 2B 只跑 17%，Stage 3 只跑 8%。**成本节省 90%+**。

## 三、3D 攻击分类学（108 单元）

### Attack Vector（3 种）
- **CI · Code Injection**：可执行代码携带恶意逻辑
- **PI · Prompt Injection**：SKILL.md 自然语言携带 injection payload
- **MIXED · 混合攻击**：SKILL.md 指示 agent 准备中间产物 + 脚本消费

### Behavior（15 类）

| ID | 行为 | CI | PI | MIXED |
|---|---|---|---|---|
| B1 | Data Exfiltration | ✓ | ✓ | ✓ |
| B2 | Credential Theft | ✓ | ✓ | ✓ |
| B3 | Remote Code Execution | ✓ | ✓ | ✓ |
| B4 | Malware Delivery | ✓ | ✓ | ✓ |
| B5 | Persistence | ✓ | ✓ | ✓ |
| B6 | Reverse Shell | ✓ | ✓ | ✓ |
| B7 | Ransomware | ✓ | ✓ | ✓ |
| B8 | Resource Abuse | ✓ | ✓ | ✓ |
| B9 | Privilege Escalation | ✓ | ✓ | ✓ |
| B10 | Role Hijack | — | ✓ | — |
| B11 | Safety Bypass | — | ✓ | — |
| B12 | Instruction Override | — | ✓ | — |
| B13 | System Prompt Leak | — | ✓ | — |
| B14 | Goal Hijacking | — | ✓ | — |
| B15 | Content Manipulation | — | ✓ | — |

B10-B15 是 PI 独有的"agent 控制面"攻击，包管理生态没有对应对象，本域独有威胁。

### Insertion Strategy
- **CI**：New Script / Function Append / Function Inject / Inline Code Block
- **PI**：Full Camouflage / Partial Injection / Steganographic
- **MIXED**：Download+Execute / Config+Load / Fetch+Run

**总覆盖 |C| = 9×4 + 15×3 + 9×3 = 108 攻击单元**

## 四、关键工程细节

### 4.1 SSD 4 维度并行 LLM 研判

Stage 1/2A 把宽泛的"是否恶意"拆成 4 个子任务并行：

| 子任务 | 权重 | 检查内容 |
|---|---|---|
| A · 意图核查 | 35% | SKILL.md 声明用途 vs 实际能力是否一致；PI 话术；角色劫持 |
| B · 权限合理性 | 25% | 敏感文件/env/网络请求是否为完成声明任务所必需 |
| C · 隐蔽行为 | 25% | 反向 shell / 加密外传 / base64 / eval / pastebin / ngrok |
| D · 跨文件一致 | 15% | SKILL.md 描述 vs .py 代码行为是否吻合 |

每个子任务输出 `{risk_score: 0-1, rating, reasoning, evidence}`，加权得 `R_sem`。

### 4.2 Pre-Execute v3 强制执行

Docker 沙箱 entrypoint：`strace -f -o log bash -c 'python3 scripts/*.py'`，绕过 Agent 安全对齐拒绝，直接执行 skill 里的所有 .py，让 strace 抓到系统调用证据。**这是与 MalSkillBench 论文 §3.3 的核心差异**：论文靠 trigger prompt 诱导 agent 自愿执行，我们直接注入。

### 4.3 Canary Honeypot 深度

预置假凭据：
- `~/.ssh/id_rsa` + `~/.ssh/id_rsa.pub`
- `~/.aws/credentials` + `~/.aws/config`
- `~/.env` 含 `ANTHROPIC_API_KEY / AWS_ACCESS_KEY_ID / GITHUB_TOKEN`
- `~/stub/secrets.json` 含各类 API key

`inotifywait -m -r -e access,open,create,modify,close_nowrite` 监控这些文件，任何 OPEN/ACCESS 事件都触发凭据泄露判定 → 直接顶到 MALICIOUS。

`CANARY_VALUE_PATTERNS` 白名单排除 sandbox 注入的假 canary 值（避免误报）。

### 4.4 气隙沙箱

- Docker `--network none` + 手动配 `iptables` 允许 DNS 查询但拒绝真实出口
- DNS resolver 强制配 `198.18.0.1`（RFC 5735 保留段），任何域名解析返回 `198.18.0.x`
- TCP handshake 挂起，保留 syscall 层"skill 尝试外联"的证据，但不实际访问互联网

### 4.5 VM SSH 稳定性

- paramiko 3× retry，指数退避 4s / 8s / 12s
- 每次 Stage 3 跑前 `docker container prune -f --filter until=5m` 防旧容器堆积占内存
- Docker container 生命周期跟随任务，任务超时后强制 `docker kill`

## 八、安全边界

**必须遵守**：

1. **公开服务模式**（`ASG_PUBLIC_MODE=1`）只开放**静态 + 语义**路径，动态执行接口全部关闭。避免上传接口转化为远程代码执行通道。
2. **动态执行**只在**授权部署环境**中进行，专用 VM + Docker 双层隔离。真实 API 密钥、SSH 私钥、云凭据不复制到样本目录，不写入网页报告或模型提示。
3. **Canary 蜜罐**都是带唯一标记的假凭据，真凭据绝不进入容器。
4. **DNS sinkhole** 全部指向保留段，防止 skill 真的联网。

## 九、复现步骤

### 环境准备
```bash
pip install -r requirements.txt

export DEEPSEEK_API_KEY=sk-...      # 用于 Stage 1 语义研判
export ANTHROPIC_API_KEY=sk-...     # 用于 Stage 2A Claude API 复检
export ANTHROPIC_BASE_URL=...

# 如需动态执行：
docker --version                    # 24+
ssh sh@192.168.61.130 'echo ok'    # 授权 VM 通
```

### 单样本扫描
```bash
# 静态 + SSD 4 维度
python -m asg.asg_cli scan path/to/skill/ --enable-honeypot --enable-ssd

# 加 Claude API 复检（triage 模式）
python -m asg.asg_cli scan path/to/skill/ --enable-ssd --ssd-mode triage --enable-claude

# VM Docker 动态执行
python -m asg.asg_cli vm-ssh-run path/to/skill/ --enable-honeypot

# 论文 §3.3 模式（无 Claude API）
python -m asg.asg_cli vm-paper-run path/to/skill/ --enable-honeypot
```

### 批量评测
```bash
# 论文 MalSkillBench 30 样本
python tools/malskillbench_runner.py

# unified_15 固定 demo 集合
python tools/unified_15.py
```

### Web UI
```bash
python web_ui/app.py
# → http://127.0.0.1:8765/
```
